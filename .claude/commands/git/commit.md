커밋 메시지: $ARGUMENTS
현재 변경사항을 분석하고 커밋을 생성해주세요.

## 커밋 메시지 작성 규칙

### 기본 규칙
1. 커밋 내용은 **한글**로 작성
2. 본문은 `-`(하이픈)으로 시작하는 리스트 형식
3. 각 항목은 구체적인 변경사항을 명확하게 기술
4. 괄호 안에 추가 설명/세부사항 포함 가능


### 형식
```
{타입}({영역}): {제목}

- {변경사항1}
- {변경사항2}
- {변경사항3}
```

### 세부 가이드라인
- **제목**: 50자 이내로 간결하게 작성
- **타입**: Conventional Commits 형식 준수
- **영역(scope)**: 변경된 주요 범위를 명확히 표시
- **본문**: 각 항목은 `-`로 시작하며 한 줄씩 작성
- 명령형 어조 ("추가" not "추가됨")
- 여러 논리적 변경사항에 대한 diff 분석
- 필요시 분할 제안
- 원자적 커밋 (단일 목적)
- 관련 없는 변경사항 분할


### 커밋 타입
- `feat`: 새로운 기능 추가
- `fix`: 버그 수정
- `docs`: 문서 수정
- `style`: 코드 포맷팅, 세미콜론 누락 등
- `refactor`: 코드 리팩토링
- `test`: 테스트 코드 추가/수정
- `chore`: 빌드, 패키지 관리 등

## 참고사항

- 스테이지된 파일이 있으면 해당 파일만 커밋
- 분할 제안을 위한 diff 분석
- **커밋에 Claude 서명 절대 추가하지 않음**
- **중요: 커밋 메시지에 "🤖 Generated with Claude Code" 및 "Co-Authored-By" 부분은 포함하지 말 것**
